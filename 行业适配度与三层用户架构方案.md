### 1.2 跨行业通用合同类型

无论哪个行业，都会用到以下 **5类通用合同**：

| 通用类型 | 占比 | 说明 |
|---------|------|------|
| 采购合同 | 100% | 所有行业都有采购需求 |
| 劳动合同 | 100% | 所有行业都需要用工 |
| 租赁合同 | 80%+ | 办公场地/设备/仓库 |
| 保密协议(NDA) | 70%+ | 商务合作基本前置 |
| 服务合同 | 90%+ | IT/咨询/外包等 |

### 1.3 行业合同的关键差异维度

从调研可以看出，行业之间的合同差异体现在 **5个核心维度**：

```
维度1 ── 合同类型差异
  建筑有「分包合同」、物流有「运单合同」、餐饮有「食材配送」
  → 每个行业有 2-3 个特有合同类型

维度2 ── 关键参数差异
  建筑关注「质保期5年」、物流关注「破损率0.5%」、租赁关注「递增率」
  → 同一个字段名，不同行业默认值/校验规则完全不同

维度3 ── 工作流差异
  建筑：总包→分包多级审批 → 跨组织协同
  租赁：押金→租期→续签 → 长期稳定型
  餐饮：供应商→配送→证照 → 高频更新型
  → 合同审批链、提醒逻辑、关联关系各不同

维度4 ── AI识别重点差异
  建筑：风险条款（工期延误/造价变更）
  租赁：金额条款（租金/押金/递增率）
  物流：责任条款（破损/延迟/保险）
  → AI解析不同行业时，提取的「合同要素」不同

维度5 ── 台账/报表差异
  建筑：按项目核算
  租赁：按资产/租客核算
  餐饮：按供应商/食材核算
  → 同一份数据，不同行业要看不同维度的报表
```

**核心场景**：

```

  拍照上传租赁合同 → AI自动提取租金/押金/租期 → 自动设置收款提醒
  → 每月1号自动提醒「租客王五的¥5,000房租还有3天到期」

场景2：个人合同归档
  邮箱自动同步（授权IMAP） → 自动识别合同附件 → AI归类 → 到期自动提醒

场景3：收支台账
  「我的应收」vs「我的应付」两个视图
  每个合同标记方向（收/支），自动生成个人台账

场景4：随手拍合同
  PaddleOCR拍照 → AI解析 → 极速入库（<10秒）
```

## 第四部分：竞争壁垒分析

### 4.1 与竞品的核心差异

```
传统CLM面临的困境：
  定制化太重 → 实施周期3-6个月 → 报价¥10万-200万 → 只有大企业用得起
  功能固定     → 不适合个人        → 不支持小企业   → 不支持行业切换

我们的解法：
  个人版免费   → 零门槛获客      → 用户习惯培养   → 向上转化
  行业插件化   → 新行业只需配置  → 几乎零开发成本  → 快速复制
  SME按年付费  → 低成本高覆盖    → SaaS规模化     → 网络效应
  大企业定制   → 高客单价        → 私有化部署     → 深度绑定
```

### 4.3 关键壁垒

1. **个人→SME→大企业的转化路径**：传统CLM不做个人市场，我们的个人版免费是流量入口

3. **个人+企业双空间**：竞品全部只有企业版，个人空间是独特价值
4. **GenUI自适应**：竞品固定菜单，我们千人千面
